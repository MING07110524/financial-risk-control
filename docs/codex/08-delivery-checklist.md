# 最终交付检查清单

## 1. 当前完成模块

- 用户认证、当前用户、退出登录
- 用户管理、角色读取、当前登录用户保护
- 风险数据录入、编辑、详情、删除保护
- 指标新增、编辑、启停，规则新增、编辑、删除
- 风险评估、历史评估快照、预警生成与处理
- 仪表盘、统计分析、最近预警、操作日志
- AI 助手占位页与 `/api/assistant/*` 占位接口

## 2. 当前已知边界

- 风险数据一旦产生评估、预警或处理历史，当前实现拒绝删除
- 指标当前不提供删除接口，测试型指标需要保持停用
- AI 助手当前固定返回 `50100 / AI assistant is disabled in V1`
- 前端构建已做拆包优化，但图表相关 chunk 仍相对较大，属于可解释工程边界

## 3. 三角色演示入口

- `risk-demo`：风险数据 -> 风险评估 -> 预警处理 -> 统计回流
- `manager-demo`：仪表盘 -> 预警详情/处理记录 -> 统计分析
- `admin-demo`：指标规则 -> 用户管理 -> 操作日志 -> AI 助手

三类账号统一密码：`demo`

## 4. 启动步骤

1. 启动 MySQL，并确认数据库连接信息可用
2. 启动后端：`cd backend && .\mvnw.cmd spring-boot:run`
3. 启动前端真实环境：`cd frontend && npm install && npm run dev`

Mock 回退环境：`cd frontend && npm run dev:mock`

## 5. 回归命令

- 后端测试：`cd backend && .\mvnw.cmd clean test`
- 前端真实构建：`cd frontend && npm run build`
- 前端 Mock 构建：`cd frontend && npm run build:mock`

## 6. 当前环境事实

- 默认前端环境：真实后端模式
- `frontend/.env.mock`：独立 Mock 回退环境
- 用户、认证、日志、风险数据、指标规则、评估、预警、统计均已持久化
- 默认运行数据库为 MySQL
- H2 仅用于测试与 CI

## 7. 常见问题排查

- **登录时报网络错误**：先确认 `backend` 已启动，并监听 `8080`
- **评估时报 400**：检查是否补齐全部启用指标值，且规则覆盖当前指标值区间
- **日志筛不到记录**：优先使用 `模块 + 操作类型 + 操作人` 组合筛选
- **Mock 按钮没出现**：确认是否使用 `npm run dev:mock` 或 `VITE_USE_MOCK=true`
- **测试型指标无法删除**：这是当前产品边界，请直接停用保留

## 8. 最终变更清单

- 后端：用户与认证改为数据库驱动，日志、风险数据、指标规则、评估、预警、统计都走真实持久化
- 前端：默认真实环境与独立 Mock 环境彻底分离，统一通过 service 层访问数据
- AI：保留页面入口和后端占位接口，不扩展真实能力
- 工程化：补充双环境脚本、GitHub Actions、构建拆包、交付文档与仓库清理规则
