# 核心用例说明

## 用例1：用户登录

- 用例名称：用户登录
- 用例编号：UC-01
- 参与者：系统管理员、风控人员、管理人员
- 前置条件：用户账号已创建且处于有效状态
- 后置条件：登录成功后进入系统首页，系统建立用户会话
- 触发条件：用户访问系统登录页面并提交账号密码

### 基本流程

1. 用户进入登录页面。
2. 用户输入用户名和密码。
3. 系统校验用户名和密码是否正确。
4. 系统校验账号状态是否有效。
5. 系统根据用户角色加载对应首页和菜单。
6. 用户登录成功并进入系统。

### 备选流程

1. 若用户名或密码错误，系统提示登录失败。
2. 若账号被停用，系统提示账号不可用。
3. 若输入为空，系统提示必填项不能为空。

### 业务规则

1. 只有有效账号才允许登录。
2. 不同角色登录后拥有不同操作权限。
3. 登录失败时不应进入系统主页。

## 用例2：风险数据录入

- 用例名称：风险数据录入
- 用例编号：UC-02
- 参与者：风控人员
- 前置条件：用户已登录系统且具有数据录入权限
- 后置条件：风险数据成功保存，状态可供后续评估使用
- 触发条件：风控人员进入风险数据管理页面并提交新数据

### 基本流程

1. 风控人员进入风险数据录入页面。
2. 系统展示数据录入表单。
3. 风控人员输入业务编号、客户信息、风险指标数据和业务说明。
4. 系统校验必填项和数据格式。
5. 系统保存风险数据。
6. 系统返回保存成功信息。

### 备选流程

1. 若必填项为空，系统提示补全信息。
2. 若业务编号重复，系统提示编号已存在。
3. 若数据格式错误，系统提示重新填写。

### 业务规则

1. 未登录用户不能录入风险数据。
2. 同一业务编号原则上不可重复录入。
3. 保存成功后的数据可参与风险评估。

## 用例3：风险指标配置

- 用例名称：风险指标配置
- 用例编号：UC-03
- 参与者：系统管理员、风控人员
- 前置条件：用户已登录并具有规则配置权限
- 后置条件：风险指标和评分规则保存成功
- 触发条件：用户进入指标管理页面并新增或修改规则

### 基本流程

1. 用户进入风险指标管理页面。
2. 系统展示已有指标与规则信息。
3. 用户新增或编辑指标名称、权重、评分规则和预警阈值。
4. 系统校验配置内容的完整性与逻辑有效性。
5. 系统保存指标规则。
6. 系统返回保存成功信息。

### 备选流程

1. 若指标名称重复，系统提示不可重复。
2. 若权重或阈值设置异常，系统提示规则不合法。
3. 若用户权限不足，系统拒绝保存操作。

### 业务规则

1. 指标名称应唯一。
2. 权重和阈值范围必须符合设定规范。
3. 新规则保存后对后续风险评估生效。

## 用例4：风险评估执行

- 用例名称：风险评估执行
- 用例编号：UC-04
- 参与者：风控人员
- 前置条件：存在可评估的风险数据，风险指标规则已配置完成
- 后置条件：生成风险评分和风险等级，并保存评估结果
- 触发条件：风控人员在评估页面发起风险评估操作

### 基本流程

1. 风控人员选择待评估的业务数据。
2. 系统读取该业务数据及对应指标规则。
3. 系统执行评分计算。
4. 系统生成风险总分和风险等级。
5. 系统保存评估结果。
6. 系统将结果展示给风控人员。

### 备选流程

1. 若无可用基础数据，系统提示无法评估。
2. 若指标规则未配置完整，系统提示规则缺失。
3. 若数据已更新，系统提示需要重新评估。

### 业务规则

1. 评估结果必须依据系统规则自动生成。
2. 风险等级按预设评分区间判定。
3. 数据变更后旧评估结果应失效或标记为待更新。

## 用例5：预警处理

- 用例名称：预警处理
- 用例编号：UC-05
- 参与者：风控人员、管理人员
- 前置条件：系统中已存在待处理的预警记录
- 后置条件：预警状态被更新，并生成处置记录
- 触发条件：用户进入预警详情页面并提交处理结果

### 基本流程

1. 用户进入预警管理页面。
2. 系统展示预警列表和状态信息。
3. 用户选择某条预警并查看详情。
4. 用户填写处理意见和处理结果。
5. 系统保存处置记录。
6. 系统更新预警状态为已处理或处理中。
7. 系统返回处理成功信息。

### 备选流程

1. 若处理意见为空，系统提示必须填写。
2. 若预警状态已完成，系统提示不可重复处理。
3. 若用户无权限处理，系统拒绝操作。

### 业务规则

1. 预警信息达到阈值后由系统自动生成。
2. 同一预警需保留完整处置过程。
3. 已处理预警不可重复提交相同处理结果。

## 用例6：风险统计查询

- 用例名称：风险统计查询
- 用例编号：UC-06
- 参与者：风控人员、管理人员
- 前置条件：系统中已存在风险数据、评估结果和预警记录
- 后置条件：系统输出统计结果和图表展示内容
- 触发条件：用户进入统计分析页面并提交查询条件

### 基本流程

1. 用户进入统计分析页面。
2. 系统展示默认统计概览。
3. 用户输入时间范围、风险等级或预警状态等条件。
4. 系统根据条件汇总数据。
5. 系统生成统计结果和图表。
6. 用户查看统计分析内容。

### 备选流程

1. 若无符合条件的数据，系统提示暂无数据。
2. 若查询条件不合法，系统提示重新输入。
3. 若用户权限受限，系统仅展示可访问范围内结果。

### 业务规则

1. 统计结果必须来源于真实业务数据。
2. 图表结果需与查询条件保持一致。
3. 不同角色查看的数据范围可以不同。
